# SECURITY.md — AIリサーチでの機密データ取扱ルール

> 適用範囲: 本ワークスペースおよび miratuku / foresight / esse-sense 等の関連リサーチプロジェクト
> ルール対応: R1-3 (機密・査読原稿を公開LLMに投入しない)
> 一次根拠: Nature / Science / Cell Press / IEEE / NeurIPS / ICML 査読ポリシー

---

## 1. 絶対禁止事項

以下を**公開 LLM API（OpenAI / Anthropic / Google / xAI 等のパブリックエンドポイント）に投入してはならない**：

- ❌ 査読中の他者の原稿（peer review under embargo）
- ❌ 共著者の合意なく未公開の研究データ
- ❌ 第三者から守秘義務付きで提供されたデータ
- ❌ 個人情報・医療情報・金融情報（GDPR / HIPAA / PCI 該当データ）
- ❌ 自社・取引先の社外秘情報
- ❌ パスワード・APIキー・SSH鍵・トークン
- ❌ 法的に開示制限のあるドキュメント（NDA、特許出願前情報、訴訟関連）

## 2. 許容される経路

機密性のあるデータを LLM 処理する必要がある場合は、以下の **いずれか** を必ず使用：

### 2.1 自己ホスト LLM
- **オンプレミス・プライベートクラウド** で動作するモデル（llama.cpp、vLLM、Ollama 等）
- ネットワーク的に外部送信されないことを確認

### 2.2 プライバシー保護モード付きエンタープライズエンドポイント
契約ベースで以下を確認：
- **訓練データへの非利用**: ユーザー入力がモデル訓練・改善に使われない契約条項
- **データ保持期間**: 30日以内、または0日（即時破棄）
- **地理的データレジデンシー**: 必要なら EU/日本国内に限定
- **監査ログ**: アクセスログが提供されること

具体例（2026 年時点で利用可能なもの）：
- Anthropic: Claude API の "no training" デフォルト + Zero Data Retention 契約オプション
- OpenAI: API 経由のデフォルトは "no training"、Enterprise Privacy オプション
- Google: Vertex AI のプライバシー設定
- AWS Bedrock / Azure OpenAI: クラウド契約に従う

> **⚠ ChatGPT.com / Claude.ai の Web UI（無料・Pro 含む）は対象外**。これらは設定で訓練利用を停止できるが、機密扱いには不十分。必ず API 経由 or Enterprise 契約。

### 2.3 完全匿名化
- 個人特定情報・組織名・固有名詞をすべて伏せ字 or 仮名化
- 抽象化レベルを上げ、復元不可能な形に
- ただし、これでも完全な保証はないため、機密性が高い場合は 2.1 を優先

## 3. 運用ルール

### 3.1 入力前のチェックリスト
LLM に何かを送る前に：

- [ ] このデータは公開済みか、または公開可能か？
- [ ] 含まれる人物・組織からの同意があるか？
- [ ] 守秘義務契約（NDA）に違反しないか？
- [ ] 個人情報・医療情報・金融情報を含まないか？
- [ ] 知的財産（特許出願前、未公開発明）を含まないか？

**1つでも No / 不明があれば送信禁止**。

### 3.2 査読原稿の取り扱い

学会・ジャーナルの査読を引き受けた原稿は **絶対に公開 LLM に投入しない**。
- Nature: 査読プロセスでの LLM 使用は守秘義務違反
- Science: 同上
- Cell Press / Elsevier: 同上
- IEEE: review 中の manuscript の public AI プラットフォーム処理は守秘義務違反
- NeurIPS / ICML: submission・コードを LLM と共有禁止

例外: 学会が明示的に LLM 使用を許容している場合のみ（その場合も査読報告書で開示）。

### 3.3 インシデント対応

機密データを誤って LLM に投入してしまった場合：

1. **即座に停止** — チャットを閉じる、APIリクエストを中断
2. **記録** — 何を、いつ、どのモデルに送信したかを記録
3. **削除依頼** — プロバイダのデータ削除リクエスト機能を利用（Anthropic / OpenAI とも提供あり）
4. **影響範囲の評価** — 関係者・組織・データ主体への通知要否を判断
5. **報告** — 組織のデータ保護責任者・上長への報告

## 4. ツールでの強制

### 4.1 Citation Agent / Verification CLI
本ワークスペースのツール (`tools/verify_citations.py`, `tools/citation_agent.py`) は **公開 API（Crossref / arXiv / OpenAlex / PubMed / HEAD）にのみアクセス**し、入力 Markdown 全文を LLM に送信することはない（Claude セマンティック検証を有効化した場合は引用箇所の主張テキスト数百字のみ送信）。

入力ファイルに機密情報が含まれる場合は、Claude セマンティック検証を **無効化** すること：

```bash
# セキュアモード（外部LLM呼び出しなし、HTTPだけ）
citation-agent 機密文書.md --no-claude

# 完全オフライン（API も呼ばない）
extract-citations 機密文書.md   # 抽出のみ、検証なし
```

### 4.2 SimpleQA / 6軸ダッシュボード
SimpleQA 等の評価ツールは **評価用クエリ** のみを送信するため、機密データは含まれない。設計上クリーン。
